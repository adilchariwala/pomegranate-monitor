### Collection: `readings`

Stores one document per sensor POST. Documents are auto-deleted after 30 days via TTL index.

**Document schema:**

| Field          | BSON Type | Required | Constraints                                    |
|----------------|-----------|----------|------------------------------------------------|
| `_id`          | ObjectId  | auto     | Auto-generated by MongoDB                      |
| `sensor_id`    | String    | yes      | 1â€“50 chars, pattern `^[a-zA-Z0-9-]+$`          |
| `temperature`  | Double    | yes      | -40.0 to 85.0 (Â°C)                             |
| `humidity`     | Double    | yes      | 0.0 to 100.0 (%)                               |
| `soil_moisture`| Double    | yes      | 0.0 to 100.0 (%)                               |
| `light_lux`    | Double    | yes      | 0.0 to 200,000.0 (lux, estimated)              |
| `timestamp`    | Date (UTC)| yes      | UTC datetime; auto-filled by backend if absent |
| `location`     | String    | no       | max 100 chars; e.g., `"living-room"`           |

**Indexes:**

| Name                        | Fields                                | Type    | Options                              |
|-----------------------------|---------------------------------------|---------|--------------------------------------|
| `sensor_timestamp_idx`      | `sensor_id ASC, timestamp DESC`       | Compound| Background, not unique               |
| `ttl_idx`                   | `timestamp ASC`                       | TTL     | `expireAfterSeconds: 2592000` (30d)  |

**Example document:**
```json
{
  "_id": "66a1b2c3d4e5f6a7b8c9d0e1",
  "sensor_id": "pomegranate-01",
  "temperature": 24.5,
  "humidity": 52.3,
  "soil_moisture": 41.0,
  "light_lux": 8200.0,
  "timestamp": "2026-04-20T14:32:00Z",
  "location": "living-room"
}
```

### Collection: `sensors`

One document per unique `sensor_id`. Auto-created on first reading from that sensor.

**Document schema:**

| Field           | BSON Type  | Required | Notes                                          |
|-----------------|------------|----------|------------------------------------------------|
| `_id`           | String     | yes      | Equals `sensor_id` (not ObjectId)              |
| `registered_at` | Date (UTC) | yes      | Set once on first POST (`$setOnInsert`)        |
| `location`      | String     | no       | Updated from each POST payload                 |
| `last_seen`     | Date (UTC) | yes      | Updated on every POST (`$set`)                 |

**Indexes:**

| Name            | Fields           | Type  | Options    |
|-----------------|------------------|-------|------------|
| `last_seen_idx` | `last_seen DESC` | Single| Background |

**Example document:**
```json
{
  "_id": "pomegranate-01",
  "registered_at": "2026-03-01T10:00:00Z",
  "location": "living-room",
  "last_seen": "2026-04-20T14:32:00Z"
}
```

**Upsert logic on POST /readings:**
```
filter:  { _id: sensor_id }
update:  {
  $setOnInsert: { registered_at: now, location: location },
  $set:         { last_seen: now }
}
options: upsert=True
```

## Plant Health Score Algorithm

Defined in `backend/app/models.py` as `compute_health_score(reading: SensorReadingCreate) -> float`.

The score is a weighted composite of four sub-scores, each 0â€“100, then clamped to [0, 100].

### Sub-score formulas

**Temperature** (weight 30%):
```
optimal: 18Â°C â€“ 35Â°C
score_temp = 100
if temp < 18: score_temp -= (18 - temp) * 10
if temp > 35: score_temp -= (temp - 35) * 10
```

**Humidity** (weight 20%):
```
optimal: 40% â€“ 60%
score_hum = 100
if hum < 40: score_hum -= (40 - hum) * 3
if hum > 60: score_hum -= (hum - 60) * 3
```

**Soil Moisture** (weight 25%):
```
optimal: 30% â€“ 60%
score_soil = 100
if soil < 30: score_soil -= (30 - soil) * 4
if soil > 60: score_soil -= (soil - 60) * 4
```

**Light** (weight 25%):
```
score_light = min(100, light_lux / 10000 * 100)
(linear: 10,000+ lux = full score)
```

**Composite:**
```
score = 0.30*score_temp + 0.20*score_hum + 0.25*score_soil + 0.25*score_light
score = max(0, min(100, score))
```
